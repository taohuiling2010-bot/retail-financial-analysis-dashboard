# 源数据说明

## 一、数据来源与合规声明

⚠️ **重要声明**

本项目使用的所有数据均为**脱敏的零售业务模拟样本数据**，**非任何真实企业的商业数据**。数据集仅用于个人学习与求职作品展示，不涉及任何商业用途、客户隐私或企业机密。

如有数据合规疑问，请通过 Issue 联系项目作者。

## 四、关键字段业务含义说明

### 4.1 三个金额字段的关系

理解 AMT、pAMT、mpAMT 三者的关系是本项目的关键：

```
吊牌总额（理论收入）= AMT + pAMT + mpAMT
    -  pAMT（促销折扣）
    -  mpAMT（会员折扣）
    =  AMT（实际营业收入）
```

举例说明：
- 商品吊牌价 ¥100，做活动打 8 折（¥80）+ 会员再 9 折（¥72）
- 实际成交价 ¥72 → AMT = 72
- 促销让利 ¥20 → pAMT = 20
- 会员让利 ¥8 → mpAMT = 8
- 吊牌总额还原 = 72 + 20 + 8 = ¥100 ✓

### 4.2 会员判定逻辑

`fct_sales.dimMemberID` 字段：
- `> 0`：会员订单（关联 dim_member 获取详细信息）
- `= 0`：非会员订单

新/老会员细分：
- 用 `dim_member.firstShoppingDate` 判断
- 首购日期在分析周期内 → 新会员
- 首购日期早于分析周期 → 老会员

### 4.3 数据完整性提示

- **样本数据**：上传至 GitHub 的 dim_*、fct_* 文件是从原始数据库导出的**前 1000 行样本**，仅供查看数据结构使用，无法直接跑出与项目演示完全一致的结果
- **完整数据**：项目演示用的看板基于完整数据集执行，故 SQL 输出 CSV（财务数据分析_sql*.csv）与样本数据可能存在差异

## 六、已知数据质量问题

项目分析过程中识别的数据质量问题（已在 SQL 中处理）：

### 6.1 优惠券混入销售明细

**现象**：`dim_goods` 中存在名为"苏泊尔压力锅优惠券"等条目，且其在 `fct_sales_item` 中有大量销售记录（QTY > 0），但 AMT、pAMT、mpAMT 全为 0。

**业务原因**：POS 系统将营销发券动作记录为"销售一个金额为 0 的商品"。

**处理方式**：在 SQL 4、SQL 6 的数据清洗层加入过滤条件：
```sql
WHERE name NOT LIKE '%优惠券%'
  AND name NOT LIKE '%赠券%'
  AND name NOT LIKE '%代金券%'
```

### 6.2 零销售 SKU 的处理

**现象**：很多商品在 `dim_goods` 中存在但在 `fct_sales_item` 中没有任何销售记录。

**处理方式**：在 SQL 4 中通过 `dim_goods CROSS JOIN shop_scope LEFT JOIN sales_agg` 让零销售商品也进入滞销分析，避免漏掉真正的滞销商品。
