# 零售连锁企业收入质量分析与折扣管控可视化项目 - 详细文档

## 一、项目背景

### 1.1 业务场景

模拟一家拥有 4 家门店、跨越华东和北京两大区的连锁零售企业，作为该企业的**财务分析师**，需要回答管理层关心的几个核心问题：

1. 当前各大区/门店的收入质量如何？折扣率是否处于健康水平？
2. 我们让出去的折扣最终去了哪里？促销让利和会员让利分别是多少？
3. 哪些门店是优质门店？哪些需要改进？怎么对标？
4. 会员体系的财务回报真的好吗？会员折扣是不是过度了？
5. 哪些商品/品类是收入支柱？哪些是高折扣依赖、需要警惕？

### 1.2 项目目标

跳出传统"销售额单一指标排名"的零售运营视角，从**财务分析师视角**重构分析框架：
- 把"销售额"还原为"吊牌额-折扣-实际收入"的财务结构
- 把"畅销/滞销"重构为"商品贡献度+折扣依赖度"二维评估
- 把"会员人数"升级为"会员经济性 ROI"评估

### 1.3 项目周期与角色

- **项目周期**：2026.02 - 2026.03
- **项目角色**：独立负责人（数据加工、模型设计、看板搭建、洞察分析全流程）
- **技术栈**：MySQL（CTE、窗口函数）、Power BI Desktop（DAX、数据建模）

## 二、数据说明

### 2.1 数据来源

模拟零售业务的 7 张核心表，包括 5 张维度表和 2 张事实表。详细字段定义见 [03_数据集/源数据_说明.md](03_数据集/源数据_说明.md)。

### 2.2 数据周期

- **本期分析周期**：2017-08-01 至 2017-08-30（30 天）
- **上期对比周期**：2017-07-01 至 2017-07-30（30 天，同口径 MTD）

### 2.3 关键字段含义

| 字段 | 含义 | 财务属性 |
|---|---|---|
| AMT | 销售金额（实际成交价 × 数量） | 收入 |
| pAMT | 促销金额（活动让利） | 收入端折扣 |
| mpAMT | 会员优惠金额（会员折扣让利） | 收入端折扣 |

⚠️ **注意**：原始数据**没有商品成本字段**，因此本项目**不涉及毛利、毛利率、净利润分析**，所有分析集中在收入端。

## 三、分析框架

### 3.1 核心收入结构模型

```
吊牌总额（理论收入）
    -  促销折扣（pAMT）
    -  会员折扣（mpAMT）
    =  实际营业收入（AMT）
```

由此推导出几个关键财务术语指标：
- **综合折扣率** = (促销折扣 + 会员折扣) / 吊牌总额
- **价格实现率** = 实际收入 / 吊牌总额（与综合折扣率互为补集）
- **促销折扣率** = 促销折扣 / 吊牌总额
- **会员折扣率** = 会员折扣 / 吊牌总额

### 3.2 时间对比框架

采用 **MTD（Month-to-Date，月初至今）同口径对比**：
- 本期：8月1日 ~ 8月30日（30天）
- 上期：7月1日 ~ 7月30日（30天）

**为何用 MTD 而非完整月份**：数据截止日为 8月30日，若用 8月整月对比 7月整月会出现"分子30天 vs 分母31天"的天数不对等，造成环比失真。MTD 是财务月中分析的规范做法。

### 3.3 多维分析层次

| 层次 | 视角 | 对应 SQL | 对应看板 |
|---|---|---|---|
| 大区 | 收入质量、折扣结构 | SQL 1 | 页面 1、页面 2 |
| 门店 | 当日运营、收入对标 | SQL 2、SQL 3 | 页面 3 |
| 商品 | 贡献度、折扣依赖 | SQL 4 | 页面 4 |
| 会员 | 客户经济性 | SQL 5 | 页面 4 |
| 品类 | 收入结构、ABC 分级 | SQL 6 | 页面 4 |

## 四、SQL 加工逻辑

### 4.1 SQL 概览

| 编号 | 名称 | 主要内容 | 关键技术 |
|---|---|---|---|
| SQL 1 | 大区收入质量与 MTD 环比 | 各大区收入、折扣率、客单价、连带率、环比 | CTE、CROSS JOIN 参数化、CASE WHEN |
| SQL 2 | 门店每日明细 | 门店日维度的收入与折扣 | INNER JOIN 子查询 |
| SQL 3 | 门店日报对比 | 当日 vs 昨日对比 | CASE WHEN 双周期聚合 |
| SQL 4 | 商品贡献度（门店视角） | 单品收入排名、滞销识别、四象限分类 | LEFT JOIN（识别零销售）、ROW_NUMBER 双排名、数据清洗（剔除优惠券）|
| SQL 5 | 会员价值分析 | 会员/非会员、新/老会员对比 | UNION ALL、订单维度聚合、首购日期判断 |
| SQL 6 | 品类结构分析 | 品类收入占比、ABC 分级、折扣依赖度 | 累计窗口函数、CROSS JOIN 总收入 |

### 4.2 关键技术点说明

**MTD 参数化**：通过 CTE `period_config` 集中管理两个周期的起止日期，避免在 CASE WHEN 里硬编码 8 处日期，便于扩展。

**滞销识别（SQL 4 核心创新）**：传统做法用销售额升序排名，会漏掉**完全没有销售记录的商品**。本项目用 `dim_goods CROSS JOIN shop_scope LEFT JOIN sales_agg` 让所有"门店×商品"组合都进入分析，零销售商品通过 LEFT JOIN 的 NULL 识别，并用三级排序（零记录优先 → 金额异常次之 → 正常销售按金额升序）确保滞销榜的业务准确性。

**数据清洗（SQL 4）**：发现 POS 系统将"优惠券"等营销工具记入销售明细，通过商品维度过滤（`name NOT LIKE '%优惠券%'`）剥离非销售记录。

**会员细分（SQL 5）**：使用 `firstShoppingDate` 判断新会员（首购日期在分析周期内）vs 老会员，UNION ALL 拼接两组对比维度。

**ABC 分级（SQL 6）**：使用累计窗口函数 `SUM() OVER (ORDER BY ... ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW)` 计算累计收入占比，按 80% / 95% 阈值划分 A/B/C 类。

## 五、Power BI 看板设计

### 5.1 数据建模

- **独立维度表设计**：除日期表外，6 张主题分析表保持独立（不互相 JOIN），分别服务于不同视觉对象——避免聚合表互相 JOIN 引发的"双重聚合"陷阱
- **门店维表**：单独构建一张包含 4 家门店的维表，作为多个分析表的统一切片器入口
- **日期表**：通过 `CALENDAR` + `ADDCOLUMNS` 构建，用于趋势分析的时间智能函数

### 5.2 DAX 度量值（25+ 项）

按主题分组：

**收入端（7 项）**：本期吊牌总额、本期实际收入、本期促销折扣额、本期会员折扣额、本期折扣总额、本期销售件数、本期订单数

**折扣率（4 项）**：综合折扣率_M、促销折扣率_M、会员折扣率_M、价格实现率

**客户经济性（3 项）**：客单价、件数连带率、SKU 连带率

**时间对比（4 项）**：上期实际收入_M、上期综合折扣率_M、收入环比、折扣率变化

**门店专属（多项）**：门店每日_实际收入、门店每日_订单数、门店每日_客单价、日报_当日收入、日报_当日折扣率、日报_收入日环比

**会员/商品/品类（多项）**：会员_订单数、会员_客单价、会员_综合折扣率、商品_实际收入、品类_实际收入 等

### 5.3 看板页面设计

#### 页面 1：收入质量驾驶舱
- 6 大 KPI 卡片（实际收入、综合折扣率、价格实现率、客单价、订单数、件数连带率）
- 综合折扣率三色预警（< 4% 绿、4-6% 橙、> 6% 红）
- 双轴趋势图（每日实际收入 + 综合折扣率）
- 大区收入对比图（本期 vs 上期）

#### 页面 2：折扣结构拆解（**核心创新页**）
- 4 大折扣率 KPI 卡片
- **收入瀑布图**：吊牌总额 → 促销折扣 → 会员折扣 → 实际收入（财务损耗路径）
- 大区折扣率对比（本期 vs 上期）
- 品类折扣率 TOP（识别高折扣依赖品类）

#### 页面 3：门店收入对标
- 4 大当日 KPI（含日环比）
- 双轴对比图（门店收入 + 折扣率）：识别"高收入低折扣"标杆店与"低收入高折扣"问题店
- 门店每日趋势折线
- 门店客单价排名

#### 页面 4：商品 & 会员价值分析
- 会员/非会员经济性矩阵（含条件格式色阶）
- 新会员 vs 老会员客单价对比
- SKU 连带率 vs 件数连带率对比
- 单品收入 TOP15
- 商品贡献度散点图（收入 × 折扣率二维分类）
- 零销售 SKU 数预警卡片
- 品类 ABC 树状图

## 六、关键洞察

### 6.1 数据治理：识别 POS 系统数据质量问题

在做单品滞销分析时发现滞销榜首商品销售额为 0，但销售记录数不少，进一步调查发现是"苏泊尔压力锅优惠券"——POS 系统将营销发券动作记入了销售明细。

**后续动作**：在数据清洗层加入商品维度过滤，剔除"优惠券"、"赠券"、"代金券"等非商品记录。

**业务启示**：做财务分析前必须先校验数据业务逻辑，不能盲信"数据库里有就是对的"。

### 6.2 会员经济性：会员体系存在"高购买力+高让利"双高现象

会员客单价 ¥60.5 比非会员 ¥45.98 高 31%，看似会员体系成功；但**会员综合折扣率 5.86% 是非会员 2.73% 的 2.14 倍**——会员是用更大的折扣换来的更高客单价。

还原到吊牌额：
- 会员吊牌客单价比非会员高 36%（验证会员选品意愿强）
- 但扣完折扣后，每个会员订单只为公司增加 ¥14.5 的实际收入

**业务启示**：会员折扣是一笔"投入"——需评估这 ¥14.5 的增量收入能否覆盖会员服务成本（积分、专属客服、会员日活动等）。这是典型的客户经济性 ROI 问题。

### 6.3 商品管控：识别 100 个零销售 SKU + 折扣依赖商品

通过 LEFT JOIN 让零销售商品也进入分析，识别全公司 4 家门店共 100 个零销售 SKU（占商品池约 18%），其中东城店占 40 个、其他三店各 20 个。

通过散点图二维分类，区分"明星商品"（高收入低折扣）与"促销依赖商品"（高收入高折扣）——后者虽然贡献收入但可能盈利能力较弱。

**业务启示**：清库决策、品类调整、定价策略需要差异化管理。

### 6.4 品类结构：80% 收入集中在烟、酒两类（A 类核心）

通过 ABC 分级分析（累计收入占比 80% / 95% 阈值），识别 A 类核心品类——这两类的折扣率也相对较高（4.7%-4.9%），处于公司平均水平，没有过度让利。

**业务启示**：品类结构高度集中，A 类品类的库存与定价策略对整体收入影响最大，需重点管理。

## 七、项目局限性

诚实地反思项目的边界：

1. **缺少成本数据**：所有分析集中在收入端，无法做毛利、净利润、ROI 等盈利能力分析。
2. **数据周期短**：仅 1 个月数据，无法做 YoY 同比、季度趋势、年度滚动等更高阶财务分析。
3. **会员分析未按门店切分**：SQL 5 是公司层面聚合，门店切片器无法联动会员视图（已通过文本注释说明）。
4. **样本数据不完整**：上传至 GitHub 的 CSV 是导出的前 1000 行，完整分析基于完整数据集执行。

## 八、可复用价值

本项目作为**零售/消费行业的财务分析模板**，其框架可复用于：

- 任何需要做"客户经济性"评估的会员体系
- 任何需要做"贡献度二维分类"的商品/品类分析

替换原始数据 + 调整字段映射后，整套 SQL + Power BI 看板均可快速移植。
